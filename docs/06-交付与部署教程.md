## 1. 三种交付形式怎么选

| 形式 | 适合阶段 | 访问方式 | 难度 |
|------|----------|----------|------|
| 本地一键启动 | 演示 / 验证价值 | `localhost` | ★ |
| 单云服务器 + Docker | 试点 / 正式使用 | `http://服务器IP:8080` | ★★ |
| 公网域名 + HTTPS | 对外正式使用、手机访问 | `https://你的域名` | ★★★ |

> FDE 建议路径：先「本地 Demo」让博主看到价值 → 再「云服务器」试点 → 稳定后加「域名 HTTPS」。

## 2. 形式一：本地一键启动（验证 Demo）

前置：本机装好 [Docker Desktop](https://www.docker.com/products/docker-desktop/)。

```bash
cd 深圳房产FDE工作流系统
cp .env.example .env
# 首次演示想看示例数据，把 .env 里 SEED_DATA 改为 true
docker compose up -d --build
```

浏览器打开 **http://localhost:8080** 即可。

停止：`docker compose down`（数据保留在 `./data`）。

## 3. 形式二：单云服务器 + Docker（推荐）

### 3.1 买服务器
- 腾讯云轻量应用服务器 / 阿里云 ECS，**2核4G** 足够，系统选 **Ubuntu 22.04**
- 安全组放行端口：`22`(SSH)、`8080`(Web)；若用 HTTPS 再放 `80`、`443`

### 3.2 装 Docker
```bash
# SSH 登录服务器后执行
curl -fsSL https://get.docker.com | sh
sudo systemctl enable --now docker
```

### 3.3 上传代码并启动
```bash
# 方式A：用 git（推荐）
git clone <你的仓库地址> /opt/fangchan
cd /opt/fangchan

# 方式B：本地用 scp 上传
# scp -r 深圳房产FDE工作流系统 root@服务器IP:/opt/fangchan

cp .env.example .env          # 按需改 WEB_PORT
docker compose up -d --build
```

### 3.4 访问
浏览器打开 **http://服务器公网IP:8080**。

### 3.5 设置开机自启
compose 里已配 `restart: unless-stopped`，服务器重启后容器会自动拉起。

## 4. 形式三：公网域名 + HTTPS

让博主用 `https://fang.xxx.com` 这样的正式域名访问（手机浏览器也安全）。

### 4.1 解析域名
在域名服务商把域名的 **A 记录**指向服务器公网 IP。

### 4.2 配置 Caddy（自动 HTTPS，零证书操作）
```bash
cp Caddyfile.example Caddyfile
# 编辑 Caddyfile，把 your-domain.com 改成你的真实域名
vim Caddyfile

# 用叠加配置启动（Caddy 自动申请 Let's Encrypt 证书）
docker compose -f docker-compose.yml -f docker-compose.https.yml up -d --build
```

打开 **https://你的域名** —— 证书由 Caddy 自动签发与续期，无需手动维护。

> 安全组需放行 `80` 和 `443`。

## 5. 数据备份与恢复

本系统数据全部在 `./data/data.db` 这一个 SQLite 文件里。

### 手动备份
```bash
sh scripts/backup.sh        # 备份到 ./backups/，自动保留最近 30 份
```

### 自动每日备份（crontab）
```bash
crontab -e
# 添加一行：每天凌晨 2 点备份
0 2 * * * cd /opt/fangchan && sh scripts/backup.sh
```

### 恢复
```bash
docker compose down
cp backups/data_20260630_020000.db data/data.db
docker compose up -d
```

> 重要：企业/客户数据无价，**务必启用自动备份**，并定期把 `backups/` 同步到另一台机器或对象存储。

## 6. 升级与回滚

### 升级（拉新代码 → 重建）
```bash
cd /opt/fangchan
sh scripts/backup.sh          # 升级前先备份
git pull                      # 或重新上传代码
docker compose up -d --build  # 重建并平滑替换
```

### 回滚
```bash
git checkout <上一个稳定版本>
docker compose up -d --build
# 如数据结构有变，按需用备份恢复 data.db
```

## 7. 常见问题排查

| 现象 | 排查 |
|------|------|
| 打不开页面 | `docker compose ps` 看容器是否 Up；安全组端口是否放行 |
| 页面打开但数据加载失败 | `docker compose logs -f backend` 看后端报错 |
| 上传 PDF 失败 | 文件过大？nginx 已设 50m，超大需调 `frontend/nginx.conf` 的 `client_max_body_size` |
| 数据丢了 | 是否误删 `./data`？从 `backups/` 恢复 |
| HTTPS 不生效 | 域名 A 记录是否指向本机？80/443 是否放行？看 `docker compose logs caddy` |

## 8. 交付清单（给客户）

交付给博主时，附上：
- [ ] 访问地址 + 使用说明（哪个菜单干什么）
- [ ] 本部署教程
- [ ] 备份策略说明（数据在哪、怎么恢复）

### 上线前安全检查清单
- [ ] `.env` 中 `SEED_DATA=false`（不带演示假数据）
- [ ] **已加登录鉴权**（当前 MVP 无鉴权，公网部署前必须补；建议给 FastAPI 加 Basic Auth / JWT）
- [ ] 客户敏感数据（手机号等）已脱敏或加密存储
- [ ] 自动备份已开启
- [ ] 用了 HTTPS（公网场景）

## 附：Docker 之外的其他交付形式

| 形式 | 说明 | 适用 |
|------|------|------|
| **Docker（本文档）** | 一键起、环境一致、易升级 | 默认推荐 |
| **裸机 systemd** | 不装 Docker，直接 uvicorn + nginx，配 systemd 守护 | 服务器极轻量/不让装 Docker |
| **你托管的 SaaS** | 你自己服务器统一托管，博主只用浏览器 | 博主完全不想碰运维 |
| **打包脚本一键安装** | 写个 `install.sh` 自动装依赖+起服务 | 半技术客户自助安装 |

> 对这个博主客户，**「你托管 SaaS」往往体验最好**：博主零运维，你统一升级；代价是数据在你这，需和客户约定数据归属与保密。
