# 03 · 数据模型设计

## 1. ER 关系总览

`Client`（客户）是协同网络的**中心节点**，所有板块都指向它。

```
                         ┌──────────┐
              ┌─────────►│  Client  │◄────────┐
              │          │ (中心)    │         │
              │          └────┬─────┘         │
        Transaction          │            Meeting
         (成交)         Contract(合同)      (会面)
              │
              ▼
         Property ◄──────── Content (内容) ──────► LandAuction
         (房源)              (自媒体)              (土拍)
                                │
                              Lead (引流线索)

         SynergyEvent (协同日志，记录所有自动联动)
```

## 2. 核心表设计

### Client（客户 / VIP）—— 协同中心
| 字段 | 类型 | 说明 |
|------|------|------|
| level | str | 普通 / 高净值 / VIP（引擎自动分级） |
| value_score | float | 客户价值分 0-100（引擎计算） |
| needs | JSON | `{"理财":bool,"保险":bool,"香港身份":bool,"子女教育":bool}` |
| tags | JSON | 标签数组，如 `["豪宅意向","改善"]` |
| interest_districts | JSON | 兴趣片区 `["福田","南山"]` |
| net_worth_estimate | float | 预估净资产（万元） |

> 用 JSON 列存 needs/tags，是为了灵活适配博主不断变化的业务标签，无需频繁改表结构。

### Transaction（成交）
| 字段 | 说明 |
|------|------|
| client_id / property_id | 关联客户与房源 |
| deal_type | 二手 / 新房 |
| deal_price / commission | 成交价 / 佣金（万元） |
| district | 福田 / 南山 / 宝中 |
| source_doc | **来源文档名**（数据可溯源） |

### LandAuction（土拍）
| 字段 | 说明 |
|------|------|
| floor_price | 楼面价（元/㎡） |
| premium_rate | 溢价率 %（≥15% 触发额外爆款选题） |

### Content（自媒体内容）
| 字段 | 说明 |
|------|------|
| category | 豪宅看房 / 土拍分析 / 区域解读 |
| status | 选题 / 拍摄中 / 已发布 |
| auto_generated | 是否协同引擎自动生成（0/1） |
| related_property_id / related_auction_id | 内容的数据来源 |

### SynergyEvent（协同事件日志）
记录引擎每一次自动联动，**可审计、可解释**：
| 字段 | 说明 |
|------|------|
| event_type | 客户升级 / 选题生成 / 调价提示 / 会面brief生成 |
| source_module → target_module | 从哪个板块协同到哪个板块 |
| description | 人话描述 |

## 3. 字段溯源设计

每条业务数据都带 `source_doc` 字段，记录它来自哪个 PDF/Word/PPT。
解决「数据散落」后最关键的信任问题：**博主能随时核对一条数据的原始出处**。

## 4. 为什么这样建模能产生协同？

把 7 个板块的数据收敛到**少数几张强关联的表**，且都挂在 `Client` 上：
- 查一个客户 → 立刻能拉出他的成交、会面、合同、需求（客户360）
- 查一个区域 → 立刻能聚合成交、在售、土拍（区域情报）
- 任何一张表新增 → 引擎能顺着外键找到该联动谁

这就是从「文件孤岛」到「关系网络」的本质升级。
