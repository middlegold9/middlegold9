# 05 · 部署与使用指南

## 1. 环境要求
- Python 3.10+
- Node.js 18+

## 2. 后端启动

```bash
cd backend

# 建虚拟环境
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 装依赖
pip install -r requirements.txt

# 灌入示例数据（会建库 + 演示协同效应）
python seed_data.py

# 启动服务
uvicorn main:app --reload --port 8000
```

访问 **http://localhost:8000/docs** 查看交互式 API 文档（Swagger）。

## 3. 前端启动

```bash
cd frontend
npm install
npm run dev
```

访问 **http://localhost:5173**。

## 4. 核心使用流程

### 流程 A：把散落文档变成数据
1. 进入「文档入库」页
2. 上传一份成交确认书 PDF / 豪宅推介 PPT / 土拍纪要 Word
3. 系统自动解析、抽取、入库，并触发协同（自动选题 / 客户评分）

或命令行批量入库：
```bash
cd backend
python -m ingestion.extractor /path/to/你的文档文件夹
```

### 流程 B：体验协同效应
1. 「VIP客户」页 → 点任意客户的「客户360」→ 看跨板块推荐
2. 「区域情报」页 → 看福田/南山/宝中的调价提示
3. 「自媒体内容」页 → 看土拍/豪宅自动生成的选题（带「自动」标签）
4. 「会面安排」页 → 点「查看Brief」→ 看自动生成的会面 brief
5. 「协同事件流」页 → 看引擎所有自动联动的审计日志

### 流程 C：生成合同
调用 API：
```bash
curl -X POST "http://localhost:8000/api/contracts/generate?transaction_id=1&contract_type=二手买卖"
```
或在「合同」页查看 seed 已生成的草稿。

## 6. 接入 LLM 增强（可选）

系统默认用规则引擎，离线可跑。需要更强能力时：
- 文档抽取更鲁棒 → 实现 `ingestion/extractor.py::extract_with_llm`
- 选题更有网感 → 实现 `services/synergy_engine.py::generate_topic_with_llm`

接入后，在对应触发点替换规则调用即可，其余架构不变。

## 7. 生产化建议（FDE 后续迭代路线）

| 阶段 | 动作 |
|------|------|
| MVP（当前） | SQLite + 规则引擎，单人可用 |
| 上量 | 切 PostgreSQL（仅改 `database.py`），加用户鉴权 |
| 智能化 | 接 LLM 做抽取/选题，接通抖音/视频号数据回流真实线索 |
| 移动端 | 微信小程序壳，随时随地录入与查看 |

## 8. 安全与合规说明
- 客户手机号等敏感信息已在示例中脱敏（`138****8888`）；生产环境应加字段级加密与权限控制。
- 文档上传走临时文件解析，避免路径穿越。
- **合同为自动生成草稿，必须经持牌人 / 律师人工校对后方可签署**，系统不构成法律意见。
- 资产配置咨询涉及理财/保险，相关建议**不构成投资承诺，投资有风险**。
